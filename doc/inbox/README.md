# Inbox — External Feature Requests & Proposals

This directory receives feature requests, proposals, and requirements from external teams, agents, or stakeholders. It is a **triage queue**, not a final home.

## Workflow

1. An external team / agent drops a document here.
2. Project owner reviews and triages.
3. Accepted items move to `../plans/` (with an implementation plan) or `../vision/` (for future exploration).
4. Rejected items are archived or deleted.

## Naming Convention

```
YYYY-MM-DD-<source>-<topic>.md
```

Example: `2026-04-17-platform-team-auth-integration.md`

## Document Template

```markdown
# Feature Request: <Title>

## Source
Team / agent / person submitting this request.

## Priority
Submitter's suggested priority (Critical / High / Medium / Low).

## Summary
One-paragraph overview.

## Problem Statement
What problem does this solve? Who is affected?

## Proposed Solution
High-level approach (if any).

## Requirements
- Requirement 1
- Requirement 2

## Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2

## Dependencies
External systems, teams, or prerequisites.

## Timeline
Desired delivery date or sprint target.
```
