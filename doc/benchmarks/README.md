# Benchmarks & E2E Verification

End-to-end test scenarios, benchmark results, and performance verification documents.

Each document either defines a **test scenario** (expected pipeline steps, commands, pass/fail criteria) or records a **benchmark run** (measured numbers from a specific date and configuration).

## Naming Convention

- `<component>-<test-type>.md` for test scenarios (e.g., `api-latency-e2e.md`)
- `YYYY-MM-DD-<description>.md` for dated benchmark runs (e.g., `2026-04-17-throughput.md`)
